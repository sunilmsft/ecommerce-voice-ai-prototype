Summary
End-to-end voice AI agent prototype for eCommerce — covers chat, voice, modality switching, human specialist handshake, quoting, and after-hours callback flows.
Prototype
🔗 Open Prototype →
Scenarios Covered
✅ PRIMARY FLOW
Customer lands on page → clicks voice icon → asks about M365 → AI recommends plan → generates quote → sends to email
📋 ADDITIONAL SCENARIOS
  • → Chat → Voice modality switch with context carry-over
  • → After-hours flow with callback scheduling
  • → Voice → Human specialist warm handshake
  • → Quote generation: happy path, email-later, skip
⬜ NOT IN PROTOTYPE
  • — Multi-language support
  • — Mobile-specific layouts
  • — Authenticated experience
Where the Prototype Ends
  • ⚙️ Voice responses use Azure TTS (pre-rendered) — production needs real-time
  • ⚙️ Quote pricing is hardcoded — production needs QuoteX API
  • ⚙️ Customer data is sample — no real CRM connection
Open Questions for Engineering
What's the right architecture for real-time context passing between chat and voice?
Can we use the existing speech SDK or do we need a custom integration?
What's the latency target for quote generation?
